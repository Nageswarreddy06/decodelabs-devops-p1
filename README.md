# 🚀 Industrial DevOps Training Portfolio

Welcome to my core DevOps infrastructure portfolio! This repository tracks my hands-on journey through industrial engineering tracks, evolving from raw file system design to automated cloud pipelines and lightweight application containerization.

---

## 📂 Case Studies: The 4 Milestone Projects

### 🖥️ Project 1: Foundational Linux Architecture & File Operations
* **Objective**: Establish a standardized, clean application workspace directory tree structure using secure multi-user permissions.
* **Implementation**: Engineered core application layouts, handled low-level terminal navigation, and configured isolated system file privileges to build a robust local directory foundation.

### 🌿 Project 2: Cloud Version Control Integration & Single Source of Truth
* **Objective**: Implement robust, trackable history states for codebase changes and build a remote tracking hub.
* **Implementation**: Initialized a local Git management configuration ledger, handled targeted file tracking tracking schemas, and established this remote GitHub workspace to enforce precise, multi-branch source control integrity.

### ⚙️ Project 3: Continuous Integration (CI) Pipeline Automation via GitHub Actions
* **Objective**: Eradicate "manual deployment error windows" by creating a fully automated software factory workspace.
* **Implementation**: Designed an infrastructure-as-code YAML automation script under `.github/workflows/pipeline.yml`. The factory instantly wakes up on every code push, spins up isolated cloud runners, installs dependencies, and runs strict architecture validation tests.

### 📦 Project 4: Microservices Containerization & Portability via Docker
* **Objective**: Solve the infamous "It works on my machine!" problem by freezing an unchangeable, isolated runtime container.
* **Implementation**: 
  * Designed a multi-layer text recipe blueprint (`Dockerfile`) using a secure, lightweight base operating system layer (`node:18-alpine`).
  * Implemented an exclusion shield (`.dockerignore`) to filter heavy environmental waste, shrinking deployment speeds.
  * Configured background process deployment flags (`-d`) and exposed a controlled external access tunnel using port mapping specifications (`-p 8080:80`).
